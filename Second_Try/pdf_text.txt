There are now TWO ways a booking can be created in the system:

 RESULT

On Accept: Employee assigns

Booking Request Business Rules
RULE
 DETAIL
 
One Active
Request
Customer can only have ONE active request (Pending or Accepted) at a time. New request allowed
only after Rejection / Expiry / Customer Cancellation.
Auto-Expiry
Request auto-expires 15 hours before the travel date+time. Status changes to Expired. Customer
notified.
Seat Selection
Default: Employee assigns seat on acceptance (standard fare). Optional: Customer may select
specific seat for premium charge — seat is NOT held during pending, only confirmed on acceptance.
Customer
Cancellation
Customer can cancel their own PENDING request at any time. Cannot cancel after acceptance.
Payment
Counter payment only. No online payment. Customer pays at branch on travel day.
Fare Display
Customer sees estimated fare breakdown on request form (based on route + bus type + age). Final
fare confirmed by employee on acceptance.
Notifications
Both in-app AND email notifications for: request received, accepted, rejected, expired, cancelled.
Request Code
Auto-generated: REQ-YYYY-XXXX format. e.g. REQ-2025-0042
